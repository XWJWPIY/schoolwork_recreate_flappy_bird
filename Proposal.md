# Game Introduction

這是一款經典的橫向卷軸街機遊戲。玩家透過點擊滑鼠或按下按鍵，控制不斷向前飛行的小鳥，藉由拍打翅膀來維持飛行高度，並閃避沿途高低不一的水管障礙物。遊戲考驗玩家的反應速度與節奏感，只要稍微失誤碰到水管或掉落地面，遊戲就會立即結束。

實作功能：

1. **基本物理與控制**：實作小鳥受到重力影響自動下墜，以及點擊時的向上跳躍。
2. **障礙物自動生成**：遊戲畫面會不斷向左捲動，並隨機生成高度不一的水管缺口供玩家穿越。
3. **碰撞偵測 (Collision Detection)**：當小鳥的邊界觸碰到水管邊緣、地面或畫面頂端時，判定遊戲結束。
4. **計分與最高分系統**：成功安全穿越一對水管即得一分，並可在本地端記錄玩家的歷史最高分數。
5. **遊戲狀態場景**：包含「首頁開始畫面 (Start Screen)」、「遊戲進行中 (Playing)」與「遊戲結束/重新開始畫面 (Game Over)」。
6. **多媒體效果**：加入背景音樂、跳躍/得分/碰撞音效，以及小鳥飛行的漸變動畫與視差捲動背景。

# Potential Problems

## 1. 物理模擬 (Physics)

- **重力加速度不自然**：若每幀直接加固定位移，小鳥下落會呈現「等速」而非「加速」。需使用 `velocity += gravity * deltaTime` 的方式讓速度隨時間累加，才能呈現真實的拋物線軌跡。
- **跳躍手感不佳**：跳躍時若直接設定固定 Y 位移，會感覺生硬。應該是將 velocity 設為一個「向上的初速度」，讓重力自然把它拉回來，才有流暢的拋物線感。
- **幀率依賴 (Frame Rate Dependency)**：若物理計算不乘以 `deltaTime`，在不同幀率的電腦上，小鳥下落速度、水管移動速度會不一致，導致遊戲難度因裝置而異。
- **小鳥旋轉角度**：原版 Flappy Bird 小鳥在上升時頭朝上、下墜時頭朝下，需要根據 Y 軸速度計算旋轉角度（`angle = f(velocity)`），否則看起來不自然。

## 2. 碰撞偵測 (Collision Detection)

- **AABB 碰撞不精確**：使用矩形框 (Axis-Aligned Bounding Box) 做碰撞偵測時，小鳥是圓形的，容易在翅膀角落產生「視覺上沒碰到但判定碰撞」的問題。可考慮縮小碰撞框或使用圓形碰撞。
- **高速穿越 (Tunneling)**：如果小鳥下落速度過快，可能在一幀之間直接穿過水管而沒偵測到碰撞。需確保碰撞檢查的頻率足夠，或使用連續碰撞偵測 (CCD)。
- **邊界判定**：小鳥飛超過畫面頂端是否應判定死亡？原版允許飛出頂部但不判死，需決定設計方向。
- **得分判定時機**：穿過水管得分的判定需準確——應該在小鳥 X 座標超過水管中心線時計分一次，要避免重複計分的 bug。

## 3. 渲染與動畫 (Rendering & Animation)

- **背景無縫銜接**：背景圖向左捲動時，兩張背景在銜接點可能出現 1px 的縫隙或重疊。需精確控制繪製位置，確保 `x2 = x1 + imageWidth`。
- **視差捲動 (Parallax Scrolling)**：背景、城市、地板若都以相同速度移動會缺乏立體感。需設定不同層的移動速率（背景最慢、地板最快）來營造深度。
- **小鳥動畫幀切換**：翅膀拍動動畫的速率需要與遊戲狀態連動——飛行時快速切換、死亡時停在固定幀。需要用計時器控制而非每幀切換。
- **Z-order 繪製順序**：背景 → 水管 → 地板 → 小鳥 → UI，繪製順序錯誤會導致元素被遮擋。

## 4. 音效 (Audio)

- **音效重疊**：快速連續按跳躍時，多個 `sfx_wing.wav` 可能同時播放導致音量爆炸或雜音。需控制同一音效的並發播放數量。
- **音效延遲**：SDL_mixer 載入音效若使用 `Mix_LoadMUS` 而非 `Mix_LoadWAV`，可能產生延遲。短音效應該用 chunk 方式載入。
- **撞擊+死亡音效順序**：碰撞時應先播 `sfx_hit`，小鳥落地後再播 `sfx_die`，兩者有先後順序而非同時。

## 5. 遊戲狀態管理 (Game State)

- **狀態轉換 bug**：Start → Playing → GameOver 三種狀態之間的轉換若沒有用狀態機管理，容易出現「死亡後還能跳躍」或「開始畫面就開始計分」等 bug。
- **重新開始的初始化遺漏**：按下重新開始後，需重置所有變數（分數、小鳥位置/速度、水管列表）。若遺漏任何一項，會出現「上一局的水管殘留」等問題。
- **輸入在錯誤狀態被觸發**：GameOver 畫面若沒屏蔽空白鍵，玩家可能誤觸跳躍。

## 6. 水管生成邏輯 (Pipe Generation)

- **缺口太小或太大**：水管上下之間的缺口大小需平衡——太小玩家根本過不去，太大又沒難度。建議先設為小鳥高度的 3~4 倍。
- **連續水管高度差過大**：若隨機生成完全不限制，可能出現前一根水管缺口在最上方、下一根在最下方的情況，玩家物理上無法反應。需限制相鄰水管缺口的最大高度差。
- **水管生成間距**：間距太短會過於密集、太長又太無聊。需根據水管移動速度調整生成頻率。
- **記憶體洩漏**：移出畫面的水管如果沒有正確移除或回收，長時間遊玩後物件數量持續增加會消耗記憶體。

## 7. 效能與相容性 (Performance)

- **圖片載入效能**：每幀重新載入圖片會嚴重拖慢效能，所有圖片應在遊戲初始化時一次性載入為 texture 並重複使用。
- **FPS 不穩定**：若遊戲邏輯、渲染沒有用固定時間步長 (fixed timestep) 或 deltaTime 做補償，畫面可能忽快忽慢。
- **視窗大小與解析度**：原版 Flappy Bird 是 288×512 的比例，需決定是否支援視窗大小調整，以及不同解析度下的素材縮放策略。
